# FMEA 综合分析表：基于 fault.template

## 说明

本表综合了以下中间分析文件：
- 00-bounds-and-layers.md: 边界与层次定义
- 01-os-layer-faults.md: OS 层故障模式

## FMEA 表格（CSV 格式）

```
一级对象,二级对象,三级对象,故障模式,可能原因,故障影响,是否中断业务,严酷度,故障模式备注,故障预防方法(预期),故障预防方法(版本现状),故障检测方法(预期),故障检测方法(版本现状),故障处理方法（预期）,故障处理方法(版本现状),故障定位方法（预期）,故障定位方法（版本现状）,改进建议,故障管理措施备注
```

## 严酷度分类说明

- **Ⅰ类（严重）**: 系统完全不可用，数据可能丢失，需要立即处理
- **Ⅱ类（较严重）**: 系统部分功能不可用，业务中断，需要及时处理
- **Ⅲ类（一般）**: 系统可降级运行，业务部分受影响
- **Ⅳ类（轻微）**: 非故障场景或轻微异常，不影响业务
