# openYuanrong datasystem 用户手册（本工作区版）

本文面向在本机使用 **`yuanrong-datasystem`（源码）** 与 **`yuanrong-datasystem-agent-workbench`（脚本与文档载体）** 的开发者：说明**环境要求**、**构建与测试**、**安装产物与 `dscli`**、以及**与官方入门一致的部署与使用要点**。

**官方入门（安装 pip、进程/K8s 部署、KV/Object/异构示例等）以 openEuler 文档为准**：  
[入门 — openYuanrong datasystem documentation](https://pages.openeuler.openatom.cn/openyuanrong-datasystem/docs/zh-cn/latest/getting-started/getting_started.html)

下文在「与官方一致」处会摘要关键命令；**细节、参数全集与更多场景请始终对照上述官方页面**。

---

## 1. 环境要求（与官方对齐）

在动手前建议自检（官方亦推荐）：

| 项 | 说明 |
|----|------|
| **Python** | 3.9、3.10 或 3.11（`pip` 安装发行版时必需；源码构建中 Python/wheel 流程亦常用） |
| **操作系统** | Linux（官方推荐 **glibc 2.34+**） |
| **架构** | x86-64（`uname -m` 为 `x86_64`） |

示例命令：

```bash
python3 --version
uname -s
uname -m
ldd --version   # 查看 glibc
```

---

## 2. 两种获得软件的方式

### 2.1 通过 PyPI 安装完整发行版（官方推荐路径之一）

适合快速体验 Python SDK、C++ SDK 与命令行工具：

```bash
pip install openyuanrong-datasystem
```

验证：

```bash
python3 -c "import yr.datasystem; print('openYuanrong datasystem installed successfully')"
dscli --version
```

完整说明见官方：[入门 — 安装](https://pages.openeuler.openatom.cn/openyuanrong-datasystem/docs/zh-cn/latest/getting-started/getting_started.html)。

### 2.2 源码编译（本仓库协作方式）

- **工程代码**：`yuanrong-datasystem`（记 **`$DS`**）  
- **构建/验证脚本与文档**：`yuanrong-datasystem-agent-workbench`（记 **`$VIBE`**）

建议在终端固定（路径改为本机绝对路径）：

```bash
export DS="/绝对路径/yuanrong-datasystem"
export VIBE="/绝对路径/yuanrong-datasystem-agent-workbench"
```

若两仓**不同父目录**，需额外：

```bash
export DATASYSTEM_ROOT="$DS"
```

**第三方依赖缓存**（避免换 build 目录就重复拉依赖）建议显式设置，见下文「构建」或 [构建产物目录与可复现工作流](verification/构建产物目录与可复现工作流.md)。

## 3. 构建（源码）

在 **`$DS`** 使用 `build.sh`（帮助信息）：

```bash
bash "$DS/build.sh" -h
```

典型首次全量构建（构建目录与安装前缀可按项目习惯调整）：

```bash
export DS_OPENSOURCE_DIR="${HOME}/.cache/yuanrong-datasystem-third-party"
mkdir -p "$DS_OPENSOURCE_DIR"

cd "$DS"
bash build.sh -t build -B "$DS/build"
```

默认安装前缀多为 **`$DS/output`**（可用 `build.sh` **`-o`** 修改）。**Client / Worker / wheel / `dscli` 落点**见 [构建产物目录与可复现工作流.md](verification/构建产物目录与可复现工作流.md)。

更细的 CMake 目标、perf、覆盖率、example 入口见 [cmake-non-bazel.md](verification/cmake-non-bazel.md)。

## 4. 安装本机构建的 Python 包与 `dscli`

源码构建成功后，通常在 **`$DS/output`** 或 **`$DS/build`** 树内可找到 **`openyuanrong_datasystem-*.whl`**（具体以构建日志与版本为准）：

```bash
find "$DS/output" "$DS/build" -maxdepth 6 -name 'openyuanrong_datasystem-*.whl' 2>/dev/null
```

安装到当前用户环境示例：

```bash
python3 -m pip install --user "$DS/output"/openyuanrong_datasystem-*.whl
command -v dscli && dscli --version
```

`dscli` 随 wheel 安装进入 `PATH` 后即可使用，与官方 `pip install openyuanrong-datasystem` 行为一致。

## 5. 测试（源码构建后）

### 5.1 务必通过 CTest 或封装脚本跑 ST

ST 二进制（如 `ds_st_kv_cache`）依赖 CMake 设置的 **`LD_LIBRARY_PATH`**。请使用：

```bash
export CTEST_OUTPUT_ON_FAILURE=1
ctest --test-dir "$DS/build" --output-on-failure
```

**不要**直接双击式运行 `build/tests/st/...` 可执行文件，否则易出现 **`libeSDKOBS.so` 缺失** 等环境问题。

### 5.2 使用 vibe 侧门禁脚本（推荐日常）

KV executor 注入相关用例与关键字审计：

```bash
bash "$VIBE/ops" test.kv_executor --skip-build "$DS/build"
```

已编过 ST 时加 **`--skip-build`** 可避免不必要的大规模重编。首次需编译目标时可去掉该参数。

### 5.3 全量与其它入口

在 **`$DS`** 仍可通过 `build.sh` 跑大类测试，例如：

```bash
bash "$DS/build.sh" -t run_cpp -l 'st*'
```

更多顺序建议见 [cmake-non-bazel.md §8](verification/cmake-non-bazel.md)。

**逐步勾选验收**（含冒烟与记录模板）：[手动验证确认指南.md](verification/手动验证确认指南.md)。

## 6. `dscli` 与集群部署（与官方一致）

以下与官方「进程部署 / Kubernetes 部署」章节一致，便于你对照复现。

### 6.1 准备 ETCD

集群管理依赖 ETCD，示例单节点（端口 **2379**）：

```bash
etcd --listen-client-urls http://0.0.0.0:2379 \
     --advertise-client-urls http://127.0.0.1:2379 &
```

### 6.2 进程方式：启动 Worker

示例在本机启动监听 **31501** 的 worker（请与客户端示例中的地址一致）：

```bash
dscli start -w --worker_address "127.0.0.1:31501" --etcd_address "127.0.0.1:2379"
```

停止：

```bash
dscli stop --worker_address "127.0.0.1:31501"
```

更多参数见官方：[进程部署](https://pages.openeuler.openatom.cn/openyuanrong-datasystem/docs/zh-cn/latest/getting-started/getting_started.html) 及站内「openYuanrong datasystem 进程部署」专题。

### 6.3 Kubernetes（摘要）

官方流程概要：

1. `dscli generate_helm_chart -o ./` 获取 Helm Chart  
2. 编辑 `./datasystem/values.yaml`（如 **`global.imageRegistry`**、**`global.images.datasystem`**、**`global.etcd.etcdAddress`** 等）  
3. `helm install openyuanrong_datasystem ./datasystem`  
4. 卸载：`helm uninstall openyuanrong_datasystem`

细节以官方 [Kubernetes 部署](https://pages.openeuler.openatom.cn/openyuanrong-datasystem/docs/zh-cn/latest/getting-started/getting_started.html) 为准。
