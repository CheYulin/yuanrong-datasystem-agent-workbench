# KV Client FEMA：时间指标与单点故障影响（粗算）

> 入口：[00-kv-client-fema-index.md](00-kv-client-fema-index.md)。短超时与 UB 感知量级差异的专项分析：[`get-latency-timeout-sensitive-analysis-5ms-20ms.md`](get-latency-timeout-sensitive-analysis-5ms-20ms.md)。

本节把可靠性表中出现的 **时间量级** 与 **客户可观测影响** 对齐，并给出单点故障下失败占比的 **工程粗算公式**（用于容量、告警与沟通口径；非严格排队论模型）。

---

## 文档中已出现的关键时间（摘自可靠性设计）

| 时间量级 | 来源 / 含义 | 对客户侧的典型影响 |
|----------|-------------|-------------------|
| **约 2s** | etcd / SDK **心跳超时**（可调），用于 Worker 故障检测与 SDK 切流触发 | **秒级窗口**内读写可能失败或切流；过后随新路由恢复 |
| **约 3s** | 故障 **检测 + 隔离** 整体窗口（与 2s 检测、约 1s 级隔离等表述一致，见整体可靠性「2s 内检测、1s 内隔离」量级） | 与下节 **剔除时间** 取齐时，可按 **2～3s** 典型值做粗算 |
| **约 100ms** | TCP **单口**切换 | 切换窗口内该路径读写报错；成功率、P99 短时劣化 |
| **20ms** | 用户侧 **RPC / Get 超时** 配置示例 | 与 UB **128ms** 硬件感知不同量级；短超时下 UB 未检测完即失败，见专项分析文档 |
| **约 128ms / 133ms** | UB **硬件侧感知**、检测 + 平面切换（128+5ms） | 与 20ms 客户超时叠加时，现象与定界需分 TCP/UB |

**影响归纳**：

- **毫秒级**（100ms、20ms）：多体现为 **成功率瞬时下降**、**P99 毛刺**、超时错误码（如 1001）。  
- **秒级**（2～3s）：多体现为 **整批实例成功率下降**、切流建链、隔离窗口内的 **2/N 类** 效应（见下）。

---

## 单点故障：2/N 与监控窗口粗算

**假设（与可靠性设计表述一致）**：某一 **单点**（例如单个 Worker 节点）故障时，该点同时参与 **全局元数据访问与数据访问路径** 中的一份份额；在节点被 **剔除 / 隔离完成前**（**秒级**，典型 **2～3s**），会有约 **2/N** 比例的 **数据访问**（含元数据与数据面相关路径的合并口径）处于失败或不可服务状态，**N** 为集群 Worker 节点总数。

在固定 **监控统计周期** \(T_{\mathrm{monitor}}\)（例如 **5s**）内，若故障持续可见时间为 **T_剔除**（取隔离/不可服务窗口，例如 **3s**），可用下列 **粗算** 将「单点窗口」映射到 **该周期内的失败占比**（用于与监控曲线、SLI 对齐）：

\[
\text{粗算失败占比} \approx \frac{2}{N} \times \frac{T_{\mathrm{剔除}}}{T_{\mathrm{monitor}}}
\]

**符号说明**：

| 符号 | 含义 |
|------|------|
| **N** | 集群中 Worker 节点数（例：**64**） |
| **2/N** | 单点故障下，相对全局元数据与数据访问的 **影响面** 工程近似（与可靠性文档中「2/n 数据读写失败」表述一致） |
| **T_剔除** | 单点从故障到被隔离、流量迁出前，客户侧持续感知的 **秒级窗口**（例：**3s**） |
| **T_monitor** | 监控或报表的 **聚合周期**（例：**5s**） |

**数值示例**（与上式一致）：\(N=64\)，\(T_{\mathrm{剔除}}=3\,\mathrm{s}\)，\(T_{\mathrm{monitor}}=5\,\mathrm{s}\)：

\[
\frac{2}{64} \times \frac{3}{5} = \frac{1}{32} \times \frac{3}{5} = \frac{3}{160} \approx 1.875\%
\]

**解读**：在 **5s** 监控桶内，若单点不可用 **3s**，且按 **2/64** 的全局影响面近似，则该桶内「与此次单点相关的」失败约占 **约 1.875%** 量级（**粗算**，实际与路由、亲和、重试、精排/召排口径有关）。  
**注意**：该式不替代线上真实成功率；用于 **故障演练、容量沟通、告警阈值** 时的数量级对齐。
