## 3. Sheet 编号与 xlsx 表名（核对时注意）

当前生成脚本 `generate_kv_client_observability_xlsx.py` 中工作表名为：

- `Sheet1_调用链路分析`
- `Sheet2_OS系统调用查表`
- `Sheet3_URMA接口查表`
- `Sheet4_性能关键路径`
- `Sheet5_定界-case查表`
- `Sheet6_URMA错误码解释`

部分 **`.md` 文件名**仍沿用早期 Sheet 序号（如 `Sheet2-URMA`、`Sheet3-TCP-RPC`），与**当前 xlsx 内 Sheet2=OS、Sheet3=URMA** 的划分**不一定同名同序**。对外打包或改 PUML 时，**以脚本生成的 xlsx 与 `README.md` 中「文件」表为准**；若发现不一致，应优先更新 Markdown 文件名或 README，而不是反推改脚本（除非有意调整列职责）。
